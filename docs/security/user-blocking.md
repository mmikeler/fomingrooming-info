# Алгоритм блокировки пользователей

Система блокировки пользователей реализована на уровне статуса аккаунта и включает два типа ограничений: полную блокировку (BANNED) и частичное ограничение (RESTRICTED).

## Статусы аккаунта (AccountStatus)

| Статус     | Описание                                                         |
| ---------- | ---------------------------------------------------------------- |
| ACTIVE     | Активный пользователь с полным функционалом                      |
| RESTRICTED | Ограниченный пользователь (может войти, но не может публиковать) |
| BANNED     | Заблокированный пользователь (не может войти в систему)          |

## Иерархия ролей

| Роль       | Уровень | Описание                                  |
| ---------- | ------- | ----------------------------------------- |
| USER       | 0       | Обычный пользователь                      |
| AUTHOR     | 1       | Автор (может публиковать без модерации)   |
| MODERATOR  | 2       | Модератор контента                        |
| ADMIN      | 3       | Администратор (управление пользователями) |
| SUPERADMIN | 4       | Суперадминистратор (полный доступ)        |

## Алгоритм блокировки (BANNED)

Полная блокировка аккаунта. Пользователь **не может** войти в систему.

### Условия блокировки:

1. **Права доступа:** Блокировать пользователей могут только ADMIN и SUPERADMIN
2. **Иерархия ролей:** Нельзя заблокировать пользователя с равной или выше ролью
   - ADMIN может заблокировать: USER, AUTHOR, MODERATOR
   - SUPERADMIN может заблокировать: USER, AUTHOR, MODERATOR, ADMIN
   - SUPERADMIN нельзя заблокировать никому
3. **Самоблокировка запрещена:** Нельзя заблокировать самого себя

### Процесс блокировки:

```
1. Проверка авторизации текущего пользователя
         ↓
2. Проверка прав (canManageUsers)
         ↓
3. Проверка иерархии ролей
         ↓
4. Проверка, что пользователь ещё не заблокирован
         ↓
5. Транзакция:
   - Обновление статуса на BANNED
   - Установка banReason (причина блокировки)
   - Установка bannedAt (время блокировки)
   - Установка bannedBy (кто заблокировал)
   - Создание записи в логе блокировок
```

#### Реализация: [`src/app/admin/actions/manageUserStatus.ts`](../../src/app/admin/actions/manageUserStatus.ts:170)

## Алгоритм ограничения (RESTRICTED)

Частичное ограничение аккаунта. Пользователь **может** войти в систему, но **не может** публиковать посты и события.

### Условия ограничения:

1. **Права доступа:** Ограничивать пользователей могут только ADMIN и SUPERADMIN
2. **Иерархия ролей:** Аналогично блокировке
3. **Состояние пользователя:** Нельзя ограничить уже заблокированного (BANNED) пользователя

### Процесс ограничения:

```
1. Проверка авторизации и прав
         ↓
2. Проверка, что пользователь не заблокирован (BANNED)
         ↓
3. Проверка, что пользователь ещё не ограничен
         ↓
4. Транзакция:
   - Обновление статуса на RESTRICTED
   - Установка restrictedReason (причина)
   - Установка restrictedAt (время)
   - Установка restrictedBy (кто ограничил)
   - Создание записи в логе
```

#### Реализация: [`src/app/admin/actions/manageUserStatus.ts`](../../src/app/admin/actions/manageUserStatus.ts:81)

## Алгоритм разблокировки

Снятие любого ограничения (BANNED или RESTRICTED) и восстановление статуса ACTIVE.

### Процесс разблокировки:

```
1. Проверка авторизации и прав
         ↓
2. Проверка, что пользователь не в статусе ACTIVE
         ↓
3. Определение типа предыдущего ограничения:
   - Если был BANNED → действие UNBANNED
   - Если был RESTRICTED → действие UNRESTRICTED
         ↓
4. Транзакция:
   - Обновление статуса на ACTIVE
   - Очистка всех полей блокировки (banReason, bannedAt, bannedBy)
   - Очистка всех полей ограничения (restrictedReason, restrictedAt, restrictedBy)
   - Создание записи в логе
```

#### Реализация: [`src/app/admin/actions/manageUserStatus.ts`](../../src/app/admin/actions/manageUserStatus.ts:264)

## Проверка блокировки при входе

При каждой попытке входа в систему выполняется проверка статуса аккаунта:

```
1. Проверка email и пароля
         ↓
2. Проверка верификации email
         ↓
3. Проверка статуса:
   - Если BANNED → выбросить ошибку с причиной блокировки
   - Иначе → разрешить вход
         ↓
4. Создание сессии с данными пользователя
```

#### Реализация: [`src/lib/auth.ts`](../../src/lib/auth.ts:60)

## Логирование действий

Все действия по блокировке/разблокировке записываются в таблицу `UserBanLog`:

| Поле          | Тип       | Описание                                                  |
| ------------- | --------- | --------------------------------------------------------- |
| id            | Int       | Уникальный ID                                             |
| userId        | Int       | ID пользователя                                           |
| action        | BanAction | Тип действия (BANNED, UNBANNED, RESTRICTED, UNRESTRICTED) |
| reason        | String?   | Причина (для блокировки/ограничения)                      |
| moderatorId   | Int       | ID модератора                                             |
| previousValue | String?   | Предыдущий статус                                         |
| createdAt     | DateTime  | Время совершения действия                                 |

#### Получение истории блокировок: [`src/app/admin/actions/manageUserStatus.ts`](../../src/app/admin/actions/manageUserStatus.ts:350)

## Проверки прав

Функции проверки прав в [`src/lib/permissions.ts`](../../src/lib/permissions.ts):

| Функция              | Описание                                                                 |
| -------------------- | ------------------------------------------------------------------------ |
| `canManageUsers()`   | Проверяет, может ли пользователь управлять пользователями (ADMIN и выше) |
| `canLogin()`         | Проверяет, может ли пользователь войти (не BANNED)                       |
| `canCreateContent()` | Проверяет, может ли пользователь создавать контент (только ACTIVE)       |

## Пример использования в компонентах

```typescript
import { useSession } from "next-auth/react";
import { canManageUsers, canCreateContent } from "@/lib/permissions";

export function AdminPanel() {
  const { data: session } = useSession();

  if (!canManageUsers(session?.user?.role)) {
    return <p>Доступ запрещён</p>;
  }

  return <UserManagement />;
}

export function CreatePostButton() {
  const { data: session } = useSession();

  if (!canCreateContent(session?.user?.status)) {
    return null; // Скрыть кнопку для ограниченных
  }

  return <button>Создать пост</button>;
}
```
