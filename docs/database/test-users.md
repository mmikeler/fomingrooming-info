# Тестовые пользователи

Для тестирования и разработки используйте следующих предустановленных пользователей:

| Email            | Пароль        | Роль       | Статус     | Описание                     |
| ---------------- | ------------- | ---------- | ---------- | ---------------------------- |
| superadmin@e.com | superadmin123 | SUPERADMIN | ACTIVE     | Суперадминистратор           |
| admin@e.com      | admin123      | ADMIN      | ACTIVE     | Администратор                |
| moderator@e.com  | moderator123  | MODERATOR  | ACTIVE     | Модератор                    |
| author@e.com     | author123     | AUTHOR     | ACTIVE     | Автор блога                  |
| user@e.com       | user123       | USER       | ACTIVE     | Обычный пользователь         |
| restricted@e.com | restricted123 | USER       | RESTRICTED | Ограниченный пользователь    |
| banned@e.com     | banned123     | USER       | BANNED     | Заблокированный пользователь |

## Использование в тестах

Эти пользователи автоматически создаются при запуске `npm run db:seed` и используются в E2E-тестах Playwright.

### Роли для разработки

- **SUPERADMIN** — полный доступ ко всем функциям, включая управление администраторами
- **ADMIN** — управление пользователями и модерацией контента
- **MODERATOR** — только модерация постов и мероприятий
- **AUTHOR** — создание и редактирование собственных постов без модерации
- **USER** — базовые права: чтение, создание постов на модерацию, регистрация на мероприятия

### Статусы для тестирования

- **ACTIVE** — стандартный активный аккаунт
- **RESTRICTED** — может войти, но не может публиковать контент
- **BANNED** — полностью заблокирован, вход запрещён

## Создание дополнительных тестовых пользователей

Для создания дополнительных пользователей добавьте их в [`prisma/seed-data/users.json`](prisma/seed-data/users.json) согласно формату, описанному в [seed-data.md](seed-data.md).
