# Просмотр отчёта

После выполнения E2E-тестов Playwright генерирует подробный HTML-отчёт.

## Расположение отчёта

Отчёт создаётся в директории `playwright-report/` в корне проекта.

## Просмотр отчёта

### 1. Открытие локально

После завершения тестов откройте файл:

```
playwright-report/index.html
```

В двойном клике на файл откроет отчёт в браузере.

### 2. Сервер отчёта

Для удобного просмотра можно запустить веб-сервер:

```bash
npx playwright show-report
```

Это откроет отчёт на `http://localhost:9323` (или другом порту).

## Содержимое отчёта

### Общая сводка

- ✅ **Passed** — количество успешных тестов
- ❌ **Failed** — количество упавших тестов
- ⏭️ **Skipped** — пропущенные тесты
- ⏱️ **Duration** — общее время выполнения

### Детали по каждому тесту

Для каждого теста отображается:

- **Название** теста
- **Статус** (✓ passed, ✗ failed)
- **Время** выполнения
- **Ошибки** (если тест упал)
- **Записи** (trace, screenshot, video)

### Attachments

При падении теста автоматически сохраняются:

- **Screenshot** — скриншот последнего состояния страницы
- **Trace** — полная трассировка действий (можно исследовать в Playwright Trace Viewer)
- **Video** — видео записи теста (если включено)

## Настройка отчёта

Конфигурация находится в `playwright.config.ts`:

```typescript
import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  // Директория для отчёта
  reportDir: "playwright-report",

  // Открывать браузер при отладке
  use: {
    headless: true, // false для отладки
  },

  // Настройки отчёта
  reporter: [
    ["html", { open: "never" }], // never, on-success, always
    ["json", { outputFile: "test-results.json" }],
  ],
});
```

## Полезные команды

```bash
# Запустить тесты и сразу показать отчёт
npm run test:e2e:ui

# Запустить тесты и открыть браузер для отладки
npm run test:e2e:debug

# Сохранитьtrace для анализа ошибок
npx playwright test --trace on
```

## Анализ ошибок

При падении теста используйте:

1. **Скриншот** — визуальное состояние на момент ошибки
2. **Trace** — полная хронология действий (network, console, DOM)
3. **Логи Playwright** — подробный вывод в консоль

Для открытия trace:

```bash
npx playwright show-trace trace.zip
```
